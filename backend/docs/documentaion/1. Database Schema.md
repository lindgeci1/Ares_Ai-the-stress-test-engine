# ARES AI — Backend Architecture: Database Schema

## Tables

| Column             | Type          | Constraints                    |
|--------------------|---------------|--------------------------------|
| id                 | uint          | PK, auto-increment             |

| password_hash      | varchar(255)  | not null                        |

| is_temp            | bool          | default `false`                 |
| access_key         | varchar(255)  | nullable, unique                |
| expires_at         | timestamp     | nullable                        |
| stripe_customer_id | varchar(255)  | nullable                        |
| subscription_tier  | varchar(50)   | default `'Free'`                |

| updated_at         | timestamp     | auto                            |

---

## Relationships

```
users 1──M documents          (user_id FK, CASCADE DELETE)
users 1──1 user_usages        (user_id PK/FK, CASCADE DELETE)
users 1──M refresh_tokens     (user_id FK, CASCADE DELETE)
users 1──M payments           (user_id FK)
users M──M roles              (via user_roles junction, CASCADE DELETE)

documents 1──M audit_reports  (document_id FK, CASCADE DELETE)
documents 1──M audio_debates  (document_id FK, CASCADE DELETE)

payments M──1 offers          (offer_id nullable FK)
```

## Migration Order (main.go)
`roles → users → refresh_tokens → user_roles → user_usages → documents → audit_reports → audio_debates → payments → offers`

Seeded on startup: default roles, default offers.
